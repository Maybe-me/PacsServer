# 重构 PACS 阅片器为 OHIF v3 风格实施方案 (1:1 像素级复刻)

本项目旨在将现有的 `pacs-viewer` 前端完全重构，在 UI 布局、视觉美学以及功能体验上 **1:1 像素级还原 OHIF Viewer v3**。此方案完全对齐 `next-phase-platform-plan.md` 中定义的“阶段 6：Cornerstone3 阅片 V1”核心目标。

## 1. OHIF 布局与视觉美学规范 (严格对齐 OHIF v3)
我们将摒弃任何多余的花哨效果，完全采用 OHIF 官方极其专业、扁平化的高信息密度 UI 风格。

- **色彩规范 (完全对齐 OHIF Theme)**: 
  - 主体背景色：极暗灰/纯黑 (`#090C10` 或 `#000000`)。
  - 面板与卡片背景色：深沉的藏青/灰蓝色 (`#05192D`)。
  - 高亮/激活状态：OHIF 经典明亮蓝 (`#00A4D9` 或 `#348CFD`)。
  - 文字颜色：主文本使用亮白色 (`#FFFFFF` 或 `#E5EDF7`)，次要文本使用暗灰色 (`#94A3B8`)。
- **页面布局结构 (严格 1:1)**:
  - **顶部视图 (Header)**: 高度固定，极简扁平。左侧显示患者信息（姓名、ID、日期），没有多余背景。
  - **工具栏 (Toolbar)**: 紧贴 Header 下方。完全复刻 OHIF 的图标组（窗宽窗位、平移、缩放、测量工具），右侧放置下拉式的布局网格选择器。
  - **左侧面板 (Left Panel - 可折叠)**: **序列浏览器 (Study Browser)**。展示序列缩略图及其元数据（设备类型 Modality、描述、图像数量）。缩略图卡片扁平且紧凑。
  - **主视口区 (Main Grid)**: 支持弹性的视口网格，无缝拼接。激活的视口具有细且亮的蓝色实线边框（非发光，纯实线）。
  - **右侧面板 (Right Panel - 可折叠)**: 用于展示测量数据、DICOM 标签或诊断信息。
- **字体与微交互**: 采用 `Inter` 或类似无衬线字体，字号偏小（12px - 14px）以保证专业软件的信息密度。按钮悬浮效果为简单的背景色变浅。

## 2. 组件重构详细计划

### [修改] `src/views/Viewer.vue`
- 使用 CSS Grid 重新设计全局布局，严格实现 `Header (Top) + Left/Right Sidebars + Main Grid` 结构。
- 移除之前所有不必要的阴影和毛玻璃代码，全面改为 OHIF 的纯色扁平填充。

### [修改] `src/components/Toolbar.vue`
- 清理现有样式，按照 OHIF 的经典工具分组进行重构：
  - **导航类**: 滚动 (Scroll)、平移 (Pan)、缩放 (Zoom)。
  - **窗宽窗位 (W/L)**。
  - **测量工具**: 长度测量 (Length)、角度测量 (Angle)。
  - **控制台**: 布局网格选择器、CINE 播放/暂停控制。

### [修改] `src/components/StudyBrowser.vue`
- 将该组件迁移至**左侧面板**。
- 缩略图卡片样式严格对齐 OHIF：上方为文字描述，下方居中显示图像，纯色背景，选中时显示 `#00A4D9` 边框。

### [修改] `src/components/DicomViewport.vue`
- 重构视口的覆盖层（Overlay），以匹配 OHIF 的四角信息展示标准：
  - 左上：患者/检查信息。
  - 右上：序列描述。
  - 左下：窗宽窗位 (W/L) 与缩放比例 (Zoom)。
  - 右下：帧数/实例数。
- 激活状态的边框改为严格的 `2px solid #00A4D9`。

### [修改] `src/composables/useViewerStore.js`
- 确保**状态集中管理**：
  - 视口网格布局状态管理。
  - 视口与序列（Series）的绑定映射关系管理。
  - 当前激活的工具及其配置状态管理。

## 3. 功能对齐与约束落地
- **Cornerstone3D 工具集成**: 确保测距、平移、缩放等工具的行为逻辑与 OHIF 完全一致。
- **预加载策略 (Preloading)**: 当选中某个序列时，遵循“预加载从第一版开始考虑”的约束，建立后台图像预加载机制。
