# 阅片 V2 需求与实现方案

## 1. 背景

当前 `pacs-viewer` 已具备基础阅片能力，包括：

- Study / Series / Instance 查询与选择
- Cornerstone3 单屏、双屏、四屏显示
- 基础工具切换、滚动、反相、重置、cine 播放
- 多帧实例渲染、坏帧跳过、黑屏诊断与兼容性提示

现阶段的问题不在于“能不能看图”，而在于整体工作区仍偏向调试型页面，距离成熟阅片工作站的体验还有明显差距。

## 2. 本轮需求目标

目标是将当前 Viewer 升级为 **Viewer V2**：

- 整体信息架构、工作区分区、交互节奏参考 OHIF Viewer 的思路
- 但不做 UI、样式、布局的逐像素复制
- 以“OHIF 风格相似版、自主实现”为目标

## 3. 目标用户

主要面向以下用户：

- 医学影像阅片人员
- PACS 管理与联调人员
- 用于样本排查、兼容性验证和演示的项目内部用户

## 4. 设计目标

Viewer V2 需要解决以下问题：

1. 当前左中右三栏偏“信息堆叠”，缺少工作站式层级
2. 顶部工具条更像调试工具集合，不像正式阅片操作区
3. Series / Thumbnail 浏览方式仍偏表格化，阅读流不够自然
4. 右侧详情区偏诊断视角，缺少面向阅片操作的上下文组织
5. 多视口状态、活动视口、降级状态还不够“显眼”和“可操作”

## 5. 实现边界

### 5.1 本期纳入

- 顶部全局工具栏重构
- 左侧患者 / 检查 / 序列导航重构
- 中心 viewport workspace 重构
- 右侧上下文面板重构
- 缩略图、序列状态、活动视口状态统一表达
- 深色主题、间距、边框、信息层级统一
- 保留并整合现有黑屏兼容性与诊断能力

### 5.2 本期不纳入

- 与 OHIF 完全一致的视觉复制
- MPR / MIP / VR 等高级重建能力
- SEG / SR / RT 叠加工作流
- 挂片协议（Hanging Protocol）完整体系
- 插件化扩展机制

## 6. 推荐界面结构

### 6.1 顶部

- 全局品牌区 / 当前检查基本信息
- 主工具按钮区
- 布局切换区
- 视口控制区
- 快捷状态区

### 6.2 左侧

- 患者 / Study 列表
- Study 下的 Series 浏览卡片
- Series 状态标签
- 快速定位缩略入口

### 6.3 中部

- 视口网格区域
- 每个 viewport 的顶部 chrome
- 当前帧 / 当前实例 / 降级状态覆盖层
- Active viewport 高亮

### 6.4 右侧

- 当前实例信息
- 当前视口状态
- 测量 / 操作上下文
- 兼容性警告与降级原因

## 7. 与当前实现的衔接原则

Viewer V2 不推翻现有能力，而是在现有基础上做结构升级：

- 保留 `useViewerStore.js` 中已有的视口状态管理
- 保留 `DicomViewport.vue` 中现有渲染、坏帧跳过、错误上报逻辑
- 保留 `viewer.js` 中当前兼容性判定与 WADO 组装逻辑
- 将当前右侧诊断信息重构为“阅片优先、诊断补充”的信息结构

## 8. 阶段性实施顺序

1. 先收口 `viewer-black-screen-followup`
2. 搭建 Viewer V2 新工作区骨架
3. 重做 toolbar 与 viewport chrome
4. 重做左侧导航与缩略图浏览
5. 重做右侧上下文面板
6. 做统一视觉收口和交互细节

## 9. 交付标准

达到以下标准可认为 Viewer V2 第一版完成：

- 页面结构已从调试台升级为工作站式阅片界面
- 单屏 / 双屏 / 四屏切换体验明显提升
- Study / Series / Thumbnail 浏览流更顺畅
- 黑屏兼容与降级信息仍然保留且更易理解
- 样式、层级、暗色主题具备统一性
